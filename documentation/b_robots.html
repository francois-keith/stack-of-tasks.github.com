Spoiler:
If you do not want to test the pouring task directly, you can run a test allowing to test joint by joints the robot. Run the following instructions (the index indicates the number of the console) to test with gazebo. 1. roslaunch sot_pr2 gazebo.launch #This starts gazebo with the pr2 in it 2. roslaunch pr2_description upload_pr2.launch roslaunch ctrl_interface vel_bridge.launch #This starts rviz and define the pr2 robot model as the current robot. 3. rosparam set robot pr2 rosrun sot_controller_wrapper start_controller #This will create several velocity controller (one by joint) and start publishing desired velocity given by the SoT. This velocity is read from the JointState value published by the sot 4. rosrun dynamic_graph_bridge run_command `rospack find robohow_sot_demo`/python/pr2_test.py 5. rosservice call /start_dynamic_graph # starts controlling the robot with the sot. 5. rosrun dynamic_graph_bridge >> help() then you can test the joints one by one using testJoint(robot, index, angle) e.g: testJoint(robot, 22, 0.2) Note that only the joints published by the ctrl_interface (aka the arms) will be taken into account. You will not be able to change the head position (unless you publish the corresponding messages). *Important* The position tasks still do not take into account the revolute joints, so it is likely that the hands of the robot turn at the beginning of the motion to reach 0. To test the pouring motion, do: 1. roslaunch sot_pr2 gazebo.launch #This starts gazebo with the pr2 in it 2. roslaunch ctrl_interface vel_bridge.launch 3. Then load everything for the pr2 robot roslaunch robohow_sot_demo robohow_pr2.launch my_robot:=pr2 kinematics:=False
